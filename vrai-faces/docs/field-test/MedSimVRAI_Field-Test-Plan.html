MedSim VRAI — Field Test Plan & Coverage Matrix

Comprehensive field validation across all core system areas · prepared for the expanded field test
Date: 2026-06-13  ·  Scope: 72 test cases across 13 core areas (33 are P1 / must-pass)  ·  Companion workbook: MedSimVRAI_Field-Test-Matrix.xlsx (execution + issue log + live coverage rollup)  ·  Goal: no gaps — every core capability gets exercised on real hardware before broader rollout.

How to use this

  1. Pre-flight first. Run scripts/preflight.sh (must be 6/6) and complete Area A (network & pairing) — it gates everything else.
  2. Work area by area in the Excel Test Matrix sheet. For each row set Result (Pass / Fail / Blocked / Not run / N-A), your initials, the date, and notes. Don't leave any P1 row "Not run" before sign-off.
  3. Coverage is visible on the workbook's Coverage sheet — a live Pass/Fail rollup per area, so gaps show at a glance.
➜ Turning issues into fixes (the Claude pipeline). When a test fails, add a row to the workbook's Issue Log — its columns are ordered to give Claude everything it needs:
  1. Fill Title · Steps to reproduce · Expected · Actual · Environment · Severity, link the Test ID, set Status = New.
  2. Filter Status = New, copy those rows (including the header), and paste into Claude Code prefixed with: "Fix these field-test issues:"
  3. Claude triages, fixes, runs the gates, and replies with the commit. Retest, then set Status = Fixed → Verified.
This makes the field test a closed loop: observe → log → paste → fixed → re-verify.
